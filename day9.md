# ☸️ Day 9 – kubectl

## 📌 Overview

Today, I learned about kubectl, the command-line tool used to communicate with a Kubernetes cluster. It allows engineers to create, view, update, and delete Kubernetes resources.

## 📚 What I Learned

- kubectl communicates with the Kubernetes API Server.
- It is the primary tool for managing Kubernetes.
- It can create, view, update, and delete resources.
- DevOps engineers use kubectl daily.

## 🎯 Common Commands

- kubectl get pods
- kubectl get deployments
- kubectl get services
- kubectl get namespaces
- kubectl describe pod <pod-name>
- kubectl apply -f file.yaml
- kubectl delete pod <pod-name>

## 💡 Key Lesson

kubectl is the command-line interface for Kubernetes. It allows engineers to manage all Kubernetes resources efficiently.

## 🚀 DevOps Insight

Most Kubernetes administration and CI/CD deployments rely on kubectl to communicate with the Kubernetes cluster.

## 📝 Day 9 Summary

Today, I learned that kubectl is the primary command-line tool for Kubernetes. It is used to manage Pods, Deployments, Services, Namespaces, and other resources by communicating with the Kubernetes API Server.
