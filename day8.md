# ☸️ Day 8 – Kubernetes YAML Manifests

## 📌 Overview

Today, I learned about Kubernetes YAML Manifests. YAML files define Kubernetes resources such as Pods, Deployments, and Services. Kubernetes reads these files and creates resources automatically.

## 📚 What I Learned

- Kubernetes resources are defined using YAML.
- YAML is human-readable.
- YAML describes the desired state of Kubernetes resources.
- Kubernetes creates resources from YAML files.
- YAML is an important part of Infrastructure as Code (IaC).

## 📄 Main Sections of a YAML Manifest

- apiVersion
- kind
- metadata
- spec

## 🎯 Benefits of YAML

- Easy to read
- Easy to edit
- Version controlled with Git
- Reusable
- Supports automation

## 💡 Key Lesson

Kubernetes uses YAML files to describe what resources should exist. Instead of manually creating resources every time, engineers define them once in YAML and let Kubernetes manage them.

## 🚀 DevOps Insight

Professional DevOps engineers write YAML manifests, store them in Git repositories, and use them to deploy applications consistently across different environments.

## 📝 Day 8 Summary

Today, I learned that Kubernetes YAML Manifests define the desired state of Kubernetes resources. YAML files are reusable, easy to manage with Git, and are the foundation of Infrastructure as Code in Kubernetes.
