# ☸️ Day 10 – First Kubernetes Hands-on

## 📌 Overview

Today, I began my first Kubernetes hands-on session. I verified my Linux environment, checked whether kubectl was installed, and tested whether my system could communicate with a Kubernetes cluster.

## 📚 What I Learned

- How to verify the Linux environment.
- How to check whether kubectl is installed.
- How to test communication with a Kubernetes cluster.
- The importance of verifying the environment before deploying applications.

## 🎯 Commands Used

- uname -a
- lsb_release -a
- kubectl version --client
- kubectl cluster-info
- kubectl get nodes

## 💡 Key Lesson

Before creating Kubernetes resources, it is important to verify that the Kubernetes tools are installed correctly and that the cluster is accessible.

## 🚀 DevOps Insight

Professional DevOps engineers always validate their environment before deploying applications. Checking tool versions and cluster connectivity helps prevent deployment failures.
